Guide d'Utilisation du Générateur

Qu'est-ce que c'est ?

Ce générateur transforme n'importe quelle partie d'échecs (au format PGN) en un mot de passe cryptographiquement sécurisé. Chaque partie unique génère un mot de passe unique et reproductible.

Le jeu d'échecs est d'une complexité astronomique. Le nombre de parties possibles est estimé à 10120, un chiffre connu sous le nom de "nombre de Shannon". C'est bien plus que le nombre d'atomes dans l'univers observable ! Cette immensité garantit que chaque partie que vous jouez est pratiquement unique, faisant d'elle une base parfaite pour un mot de passe que vous seul pouvez recréer.

Avantage : Vos mots de passe sont basés sur vos parties d'échecs mémorables !

Méthode 1 : Coller un PGN existant

Si vous avez déjà une partie au format PGN :

  • Collez votre PGN dans la zone de texte
  • Ajustez la longueur du mot de passe (8-32 caractères)
  • Cliquez sur "Generate" pour créer votre mot de passe
  • Optionnel : Cliquez "Load PGN to Board" pour visualiser la partie

Méthode 2 : Jouer une partie sur le board

Pour créer une nouvelle partie directement :

  • Utilisez le board d'échecs ci-dessus pour jouer vos coups
  • Déplacez les pièces en cliquant et glissant (drag & drop)
  • Le PGN se génère automatiquement dans la zone de texte
  • Cliquez "Generate" quand vous êtes satisfait de votre partie

Navigation dans une partie

Une fois qu'une partie est chargée sur le board :

  • Utilisez les boutons "Start", "Previous", "Next", "End" pour naviguer
  • Observez les coups joués étape par étape
  • Le compteur de coups s'affiche en temps réel

Sécurité & Reproductibilité

Points importants à retenir :

  • Déterministe : La même partie PGN génère toujours le même mot de passe
  • Cryptographiquement sécurisé : Utilise SHA-256 pour le hashing
  • Unique : Chaque partie différente = mot de passe différent
  • Mode discret : Option pour masquer le mot de passe à l'écran

L'algorithme de génération en détail

La transformation de votre partie d'échecs en mot de passe suit un processus rigoureux et transparent :

  1. Nettoyage du PGN : Seule la séquence brute des coups est conservée (ex: e4 e5 Nf3 Nc6...). Les en-têtes (joueurs, date), les commentaires, les numéros de coups et le résultat sont retirés. Cela garantit que seul le déroulement du jeu influence le mot de passe.
  2. Hashing SHA-256 : La séquence de coups nettoyée est convertie en une empreinte numérique unique de 256 bits (32 octets) via l'algorithme de hachage sécurisé SHA-256. Cette opération est à sens unique : il est cryptographiquement impossible de retrouver la partie à partir de son empreinte.
  3. Construction du mot de passe : L'empreinte de 32 octets est utilisée comme une source d'aléa déterministe pour choisir les caractères :
    • Pour garantir la complexité, les 4 premiers octets sont utilisés pour générer une majuscule, une minuscule, un chiffre et un caractère spécial.
    • Les octets suivants sont utilisés pour générer le reste des caractères du mot de passe jusqu'à atteindre la longueur désirée.
  4. Mélange et finalisation : Les caractères sont ensuite mélangés (toujours de la même manière pour une même partie) pour que les premiers caractères ne soient pas prévisibles. Le mot de passe est finalement tronqué à la longueur que vous avez choisie.

Ce processus garantit que même une modification mineure dans la partie (un seul coup différent) produit un mot de passe complètement différent.

Conseils d'utilisation

  • Sauvegardez vos parties PGN importantes dans un endroit sûr
  • Utilisez des parties mémorables pour vous souvenir du mot de passe
  • Variez la longueur selon vos besoins de sécurité
  • Le mode discret est recommandé dans les espaces publics
